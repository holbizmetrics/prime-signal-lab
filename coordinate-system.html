🧭 The Coordinate System

Why knowing one factor is like having a decryption key

🎯 The Core Insight

Multiplication destroys the coordinate system.
Each bit of N depends on ALL bits of p and q (through carries).
Knowing one factor RESTORES the coordinate system — makes bits interpretable.

❌ Without p (No Reference Frame)

Each bit of N encodes information from ALL input positions

N = 8633

Dependency Map (what each output bit depends on)

The Problem: To find q[k], we'd need to know which bits of N "came from" position k. But carries have mixed everything together.

✅ With p (Reference Frame Restored)

Division is just repeated comparison against p

p = 97 (our reference)

Division: q = N / p

The Key: Each bit of q is determined by ONE comparison:
"Is remainder ≥ p × 2^k?"

This is LOCAL — no need to know other bits of q first!

🔍 Fermat's Method: Searching from √N

Your intuition: start at √N and search outward for factors

📐 The Mathematical Truth

For N = p × q where p ≤ q: √N is the geometric mean of p and q S/2 = (p+q)/2 is the arithmetic mean For balanced factors: √N ≈ S/2 ≈ p ≈ q For unbalanced factors: p << √N << q Fermat searches from √N upward for a = (p+q)/2 Then checks if a² - N = ((p-q)/2)² is a perfect square Search distance = a - √N = (p+q)/2 - √(pq) = (√p - √q)²/2 × (1/√p + 1/√q)

Why Search Distance Scales with Imbalance

Imbalance (q/p) Example √N (p+q)/2 Search Steps Verdict
The Calibration:
• Balanced factors → Small search → Fermat works fast ✓
• Unbalanced factors → Large search → Fermat fails (exponential) ✗

This is EXACTLY the calibration we discovered: error scales with imbalance!

Bottom Line:

• Fermat works when factors are balanced (but then simple trial division also works quickly)
• Fermat fails when factors are unbalanced (search space explodes)
• RSA chooses factors that are close but not too close — in the sweet spot where:
    - Close enough that Fermat search is huge
    - Far enough that special methods don't apply

The √N starting point is optimal given no information, but "optimal with no information" still means exponential search for unbalanced factors.
