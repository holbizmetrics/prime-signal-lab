A visual journey into why integer factorization is hard

The Question

Given N = p × q, can we recover p and q?

Multiplication: p, q → N (easy, O(n²)) Factorization: N → p, q (hard, no polynomial algorithm)

This asymmetry secures RSA. But why is it hard?

The Core Insight

Multiplication destroys locality through carry propagation.

Each output bit becomes a function of ALL input bits. This "carry avalanche" creates non-local dependencies that make factorization require global search.
The carry avalanche is not a bug. It is the feature that makes RSA work.

The Cruel Irony

S = p + q contains 100% of the information needed to factor!
Given N and S: p, q = (S ± √(S² - 4N)) / 2
But we don't have S. We only have N.
